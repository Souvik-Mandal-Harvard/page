Every project in LS100 follows the same arc — frame a question, learn the tools, collect data,
analyze it, and communicate the result. This page maps the whole route; you'll only ever focus
on your next step.

![The LS100 research learning path](_static/LS100_Learning-Path_Roadmap.png)

## Jump to your starting point

- **I'm new to coding** → start with [Module 00B · Python Fundamentals](../LS100_00B_Python-Fundamentals/index.md), Notebook 01.
- **I already know Python** → skim 00B, then pick your data modality below.
- **I work with video** → [Module 01A · Video Data](../LS100_01A_Video-data/index.md).
- **I work with audio** → [Module 01B · Audio Data](../LS100_01B_Audio-Data/index.md).
- **I work with digital/tabular data** → [Module 02 · Data Science & Statistics](../LS100_02_Data-Science_Stats/index.md).

## The milestone path

1. **Frame your study** — [Research Guide 01](../LS100_00A_Research-Plans-And-Proposals/index.md): turn an interest into a researchable question.
2. **Build Python fluency** — [Module 00B](../LS100_00B_Python-Fundamentals/index.md), Notebooks 01–05.
3. **Choose your data modality** — [Video (01A)](../LS100_01A_Video-data/index.md) or [Audio (01B)](../LS100_01B_Audio-Data/index.md), and run the processing pipeline. *(This is the one fork in the path — both branches rejoin at analysis.)*
4. **Analyze your outcomes** — [Module 02](../LS100_02_Data-Science_Stats/index.md), using the statistics guide and the decision-tree poster to pick the right test.
5. **Communicate your findings** — [Research Guide 02 · Art of Communicating Science](../LS100_03_Communication-and-Presentation/index.md).

:::{tip}
Stuck on a term? See the [Essential Python Terminologies guide](../LS100_00B_Python-Fundamentals/index.md).
Unsure which test to use? See the [statistics decision tree](../LS100_02_Data-Science_Stats/index.md).
:::
